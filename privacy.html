Privacy Policy

Last updated: June 29, 2026

Amaliometria Shortsmith is designed as a local tool. This policy explains what information may be processed when you use it with third-party publishing platforms.

Information processed locally

Shortsmith may process local video files, transcripts, generated clips, captions, titles, hashtags, summaries, timestamps, and publishing metadata. These files are stored on the user's machine unless the user chooses to upload or publish them.

Third-party platform access

When you connect a platform account, Shortsmith may use OAuth tokens or API credentials that you provide locally. These credentials are used to call official platform APIs for actions such as validating an account, uploading media, checking upload status, or publishing a post.

Temporary media hosting

Some platforms require a public HTTPS media URL before publishing. In those cases, Shortsmith may upload a generated video file to a configured asset host so the platform can fetch it. You control which asset host is configured and which media is uploaded.

Data sharing

Shortsmith sends data to third-party platforms only when needed for user-authorized features, such as publishing a video or validating account access. The app does not sell personal data.

Data retention

Local jobs, rendered clips, metadata, and temporary files remain on the user's machine until deleted by the user. Temporary hosted videos should be cleaned up when they are no longer needed.

Security

API keys, access tokens, refresh tokens, and SSH keys should be stored in local environment files or secure local storage and must not be committed to public repositories. Users are responsible for protecting their own credentials.

User control

You can revoke platform access from each platform's account or developer settings. You can also delete local files, clear tokens from your environment file, and remove temporary hosted media.

Contact

For privacy questions, contact the maintainer through the GitHub repository.
